# Contributing to Taskio

Thanks for wanting to contribute to Taskio! We're happy you're interested.

## How to contribute?

1. **Fork the project** on GitHub
2. **Clone your fork** locally
3. **Create a branch** for your feature/fix:
   ```bash
   git checkout -b feature/my-awesome-feature
   ```
4. **Code your feature**
5. **Commit your changes** with a clear message
6. **Push** to your fork
7. **Open a Pull Request** and we'll review it together!

## Simple conventions

### Branch naming

- `feature/description` - New features
- `fix/description` - Bug fixes
- `docs/description` - Documentation

### Code

We try to follow **PSR-12** standards for PHP but it's not critical if it's not perfect. The basics:
- 4 spaces for indentation in PHP
- Clean and readable code
- Keep it simple

### Commits

Write clear commit messages:
```bash
git commit -m "Add drag and drop functionality"
git commit -m "Fix card display bug"
```

## Tests

Before pushing, make sure existing tests still pass:

```bash
php bin/phpunit
```

## Pull Requests

No need for complicated templates! Just explain:
- What you did
- Why you did it
- Anything specific to test

## Need help?

Don't hesitate to open an issue if you have questions or get stuck. We're here to help each other!

## Code of conduct

Be nice, respectful, and remember we're all learning. No judgment, just collaboration.

---

**In short**: Fork → Branch → Code → Commit → Push → PR.  
### And we're here if you have questions !
